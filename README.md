Висновки щодо ефективності алгоритмів пошуку підрядка
Аналіз швидкодії для Статті 1
Найшвидшим алгоритмом для першого тексту виявився Боєра-Мура. При пошуку існуючого підрядка ("структурі даних") він витратив 0.0012 сек, а при пошуку відсутнього ("космос") — 0.0070 сек. Алгоритм Кнута-Морріса-Пратта відпрацював значно повільніше. Алгоритм Рабіна-Карпа продемонстрував найнижчу швидкість (0.0067 сек та 0.0286 сек відповідно).

Аналіз швидкодії для Статті 2
Результати на другому тексті повністю підтверджують тенденцію. Алгоритм Боєра-Мура залишається найшвидшим (0.0019 сек для наявного слова "Аналіз" і 0.0099 сек для відсутнього "космос"). Алгоритм Кнута-Морріса-Пратта займає середню позицію, а Рабін-Карп є найповільнішим, показавши час 0.0410 сек при повному проходженні тексту без збігів.

Загальний висновок
Алгоритм Боєра-Мура є абсолютним лідером. Його ефективність є найвищою для обох текстів незалежно від того, чи існує підрядок у тексті. Механізм зсувів дозволяє пропускати значні частини тексту без посимвольного порівняння, що мінімізує час виконання.

Алгоритм Кнута-Морріса-Пратта демонструє середні показники. Завдяки префікс-функції він уникає повторних перевірок уже пройдених символів, але його кроки зсуву менші, ніж у Боєра-Мура, тому загальний час виконання вищий.

Алгоритм Рабіна-Карпа є найповільнішим для даних задач. Постійний перерахунок поліноміального хешу для кожного "вікна" тексту потребує значних обчислювальних ресурсів, що робить цей алгоритм неефективним для базового пошуку одного підрядка порівняно з іншими методами.
